# Impeccable — SlugBase design context

Persistent notes for AI-assisted design and frontend work. **Refine the sections below** after product-owner input (see open questions in chat).

## Design Context

### Users

**Who:** People who manage many links—individuals, small teams, and operators who self-host or use SlugBase Cloud. Includes multilingual audiences (broad i18n coverage in the app).

**Context:** Repeated, task-focused sessions: finding a bookmark, organizing folders/tags, sharing, optional short redirects (`/go/:slug`), admin and setup. Power-user affordances matter: global search (Ctrl+K), compact list vs cards, bulk actions, density controls.

**Job to be done:** Keep links organized, retrievable, and shareable under *their* rules—without surrendering structure or language to a generic consumer bookmark tool.

### Brand Personality

**Voice (from product copy):** Direct and ownership-focused—“your links, your structure, your language, your rules.” Technical credibility (self-host, OIDC, API tokens) without being cold.

**Visual identity (from implementation):** Navy-tinted neutrals with a strong orange primary (`#E64100` / `hsl(17 100% 45%)`). Dark surfaces anchor on deep navy (`#0b162A`); light mode uses a cool gray-navy-tinted canvas and white cards. Icons: Lucide. Brand marks in `frontend/public/` (`favicon.svg`, `slugbase_icon_*.svg`).

**Suggested 3-word personality (confirm or replace):** Capable, sovereign, structured.

**Emotional goals (inferred):** Confidence and control first; calm, efficient workflows; trust for data that “lives with you.”

### Aesthetic Direction

**Tone:** Productivity-forward web app—shadcn-style tokens, `0.5rem` radius, Tailwind + Radix patterns. Not marketing-landing minimalism; information density and clear hierarchy for lists, tables, and dialogs.

**Theme:** Light and dark are both first-class (`next-themes`, `.dark` token set in `frontend/src/index.css`).

**Typography:** System UI stack in `index.css` (no custom brand typeface wired in yet—room to define one).

**Motion:** `tailwindcss-animate` is available; core and cloud `index.css` include a global `prefers-reduced-motion: reduce` baseline (near-zero animation/transition duration). Still avoid gratuitous motion in new UI.

**References / anti-references:** *Not yet captured from product owner—add explicit examples after discussion.*

### Design Principles

1. **Token-first surfaces** — Prefer semantic Tailwind tokens (`bg-background`, `text-foreground`, `primary`, `muted-foreground`, sidebar tokens) and CSS variables in `index.css` over ad hoc hex/RGB in components.

2. **Parity across themes** — Any new UI must read correctly in light and dark; check contrast on navy and orange pairings (primary foreground swaps by theme).

3. **Accessible by default** — Build on Radix primitives; keep visible focus styles (`outline` uses `--ring`); don’t rely on color alone for state.

4. **Efficiency for collectors** — Favor scannable lists, clear metadata (folders, tags, usage), fast paths (command palette, keyboard-friendly controls), and respectful density options.

5. **i18n-ready copy** — User-visible strings go through the existing i18n pipeline; avoid hard-coded English in UI.

6. **Self-hosted / cloud neutrality in core UI** — Core open-source UI should not assume SaaS-only concepts; cloud-specific affordances stay in the cloud layer (workspace rules).

## SlugBase Cloud — marketing decisions (Phase 0, locked)

*Phase 0 marketing shell decisions for the hosted product at [slugbase.app](https://slugbase.app). Core open-source UI stays unchanged.*

### P0-T1 — Marketing tone

**Chosen:** **Product-led editorial** — stay close to the app’s clarity and token system (utilitarian baseline), but **break card-grid monotony** in at least one major landing section (asymmetric band, split layout, or editorial block). No gradient-text or neon “AI slop”; differentiation comes from **layout rhythm** and **typographic hierarchy**, not decorative effects.

### P0-T2 — Header / footer IA

**Header (max 5 primary items):** Pricing · Contact · Log in · Sign up.

**Footer (new in Phase 2):** Terms · Privacy · Imprint, plus a short copyright line. Legal links leave the top bar to reduce cognitive load.

### P0-T3 — Marketing web font

**Chosen:** **None for now.** Marketing and app keep the **system UI stack** until a deliberate Phase 4 pass. If added later: load only on marketing routes, `font-display: swap`, metric-matched fallback, document here — prefer distinctive families over Inter/Roboto (e.g. pairing ideas in frontend-design typography reference).

### P0-T4 — Marketing H1 type scale

| Page type | Classes |
|-----------|---------|
| Landing hero | `text-5xl sm:text-6xl lg:text-7xl font-bold` |
| Interior marketing (Pricing, Contact) | `text-3xl sm:text-4xl font-bold` |
| Legal (Terms, Privacy, Imprint) | `text-2xl sm:text-3xl font-semibold` |
| Section H2 (Landing, Pricing) | `text-xl font-semibold` (preserve) |
